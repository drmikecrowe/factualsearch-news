<task type="auto">
  <name>Task 2: Create Privacy page rendering MDX content</name>
  <files>src/pages/privacy.astro</files>
  <action>
Create the Privacy page at `src/pages/privacy.astro` that renders MDX content from the content collection.

The page should:
1. Import `getEntry` and `render` from `astro:content`
2. Import BaseLayout from `../layouts/BaseLayout.astro`
3. Fetch the privacy entry using `getEntry('pages', 'privacy')`
4. Render the entry using `render(entry)` to get `{ Content }`
5. Handle the case where entry is not found (redirect or error)
6. Pass SEO metadata from entry.data to BaseLayout

Structure:
```astro
---
import { getEntry, render } from 'astro:content';
import BaseLayout from '../layouts/BaseLayout.astro';

const entry = await getEntry('pages', 'privacy');

if (!entry) {
  return Astro.redirect('/404');
}

const { Content } = await render(entry);
---

<BaseLayout title={entry.data.title} description={entry.data.description}>
  <main id="privacy-page" class="container mx-auto px-4 py-8 max-w-3xl">
    <article class="prose prose-lg dark:prose-invert">
      <Content />
    </article>
  </main>
</BaseLayout>
```

IMPORTANT:
- Use `getEntry` not `getCollection` (we want a single entry)
- The `render()` function returns `{ Content, headings }` - destructure Content
- Use Tailwind typography plugin classes (`prose`) for MDX content styling
- The title and description come from the MDX frontmatter (entry.data)
  </action>
  <verify>
    ```bash
    # Verify privacy page exists with content rendering
    test -f src/pages/privacy.astro && grep -q "getEntry.*pages.*privacy" src/pages/privacy.astro && grep -q "Content" src/pages/privacy.astro && echo "Privacy page valid"
    ```
  </verify>
  <done>Privacy page exists at src/pages/privacy.astro with MDX content rendering via getEntry/render, BaseLayout wrapper, and SEO metadata from frontmatter.</done>
