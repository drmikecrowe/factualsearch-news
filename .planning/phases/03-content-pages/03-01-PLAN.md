<task type="auto">
  <name>Task 1: Create Astro 5 content configuration</name>
  <files>src/content.config.ts</files>
  <action>
Create the Astro 5 Content Layer configuration file at `src/content.config.ts` (note: this is the NEW location for Astro 5, NOT `src/content/config.ts`).

The configuration should:
1. Import `defineCollection` and `z` from `astro:content`
2. Import `glob` from `astro/loaders`
3. Define a "pages" collection using glob loader pointing to `./src/content/pages/**/*.mdx`
4. Define Zod schema with required fields:
   - `title`: string (required)
   - `description`: string (required)
   - `lastUpdated`: coerce.date().optional()
5. Export the collections object

IMPORTANT: Use the Astro 5 Content Layer API pattern, not the legacy `src/content/config.ts` location. The loader property is required in Astro 5.

Example structure:
```typescript
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const pages = defineCollection({
  loader: glob({ pattern: "**/*.mdx", base: "./src/content/pages" }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    lastUpdated: z.coerce.date().optional(),
  }),
});

export const collections = { pages };
```
  </action>

  </verify>
  <done>Content Layer configuration exists at src/content.config.ts with pages collection, glob loader, and Zod schema for title/description/lastUpdated fields.</done>
